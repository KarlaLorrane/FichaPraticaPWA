1 - O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

Res.: De forma simples, podemos dizer que o commit é a ação de salvar as alterações do código, 
push é a ação de enviar as alterações para o repositório e pop é a ação de desfazer as alterações.


2 – Quais sáo as vantagens de usar Branchs num repositório git?

Res.: As vantagens de usar branchs em um repositório git são:
Organização, poder trabalhar em paralelo com outros coworkers, manter o histórico, testar novas funcionalidades sem afetar o código principal, 
manter o código principal sempre funcional.

3 – Nos Merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em
que consistem e como podem ser resolvidos esses conflitos?

Res.: Conflitos são situações onde o git não consegue resolver sozinho, pois existem alterações no mesmo ficheiro em linhas diferentes.
Como resolver? Abrir o ficheiro em questão e resolver manualmente as alterações, escolhendo qual alteração deve ser mantida.

4 – “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as
diferenças entre eles?

Res.: Clone é uma cópia exata do repositório original, enquanto o fork é uma cópia do repositório original, 
mas que pode ser alterada sem afetar o repositório original.
